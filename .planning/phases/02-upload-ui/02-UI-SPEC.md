**Accent reserved for:**
- The 3px bottom-left border on file list rows (existing pattern)
- The 4px left border on confirmation card

- Input field focus border ring
- The animated red bar above the section heading

Accent is NOT used on: body text, labels, placeholders, inactive input borders, progress bar track (only progress bar fill).

## Component Inventory

Phase 2 wires the existing visual prototype to real behavior. The following components need to be built or wired:

### ShareStory (existing shell — needs wiring)
- **File:** `app/page.tsx` — extract to `app/components/ShareStory.tsx`
- **Current state:** Visual prototype with fake `setSubmitted(true)` on submit
- **Phase 2 deliverable:** Replace fake submit with XHR upload pipeline

### UploadZone (sub-component)
- **Behavior:** Drag-and-drop + click-to-browse, accepts `video/*,audio/*`
- **States:**
  - `idle` — dashed border `#9CA3AF40`, background `#1F1F1F`, shows icon + label
  - `drag-active` — dashed border `#B91C1C`, background `#B91C1C08`, transition 0.3s
  - `file-selected` — drop zone collapses to file list; drop zone stays visible for adding more files
  - `uploading` — drop zone disabled (pointer-events: none, opacity 0.5)
  - `error` — dashed border `#B91C1C`, error message visible below zone
- **Mobile:** `<input type="file" accept="video/*,audio/*">` must be tappable on iOS Safari. The hidden input receives focus via the visible tap target. Do NOT rely on drag-and-drop on mobile.

### ProgressBar (new component)
- **Appearance:** Full-width horizontal bar, height 4px, track color `#2A2A2A`, fill color `#B91C1C`
- **Location:** Immediately below the drop zone / file list, above the submit button
- **States:**
  - `hidden` — display: none when no upload in progress
  - `active` — fills left-to-right proportional to XHR `progress` event percent
  - `complete` — fills to 100%, held briefly (300ms), then transitions to confirmation
- **Percentage label:** Source Sans 3, 13px, weight 400, color `#9CA3AF`, right-aligned above or beside the bar
- **XHR requirement (UPLD-03):** Use `XMLHttpRequest` with `upload.onprogress` event — NOT fetch. This is locked by the requirements.

### FileListRow (existing shell — needs state)
- **Appearance (existing):** `background: #2A2A2A`, `padding: 10px 16px`, `borderLeft: 3px solid #B91C1C`
- **Additions for Phase 2:**
  - Upload status indicator: pending (no icon) → uploading (progress text "Uploading…") → done (checkmark via CSS, color `#9CA3AF`) → failed (red "Failed" text + inline retry link)
  - Remove button: `×` character, 14px, color `#9CA3AF`, right side, only visible on hover (desktop) / always visible (mobile)

### ErrorBanner (new component)
- **Trigger conditions:**
  - Wrong file type selected (before upload starts)
  - XHR upload failure (non-2xx response or network error)
- **Appearance:** `background: #7F1D1D20`, `border: 1px solid #B91C1C`, `padding: 12px 16px`, Source Sans 3 14px weight 600, color `#B91C1C`
- **Location:** Between file list and submit button
- **Dismissal:** Disappears when user selects a new file or clicks retry

## Accessibility

- The hidden `<input type="file">` is triggered by clicking the drop zone div. The drop zone div must have `role="button"` and `tabIndex={0}` with an `onKeyDown` handler for Enter/Space to maintain keyboard accessibility.
- The progress bar must have `role="progressbar"`, `aria-valuenow={percent}`, `aria-valuemin={0}`, `aria-valuemax={100}`.
- Error messages must be announced to screen readers via `role="alert"` on the ErrorBanner container.
- Touch target minimum: 44px height on all interactive elements. The file input hidden element proxied through the drop zone satisfies this when the zone is at least 44px tall.
