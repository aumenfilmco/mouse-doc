Create `app/hooks/useFileUpload.ts` implementing the XHR upload state machine hook. The hook must:

1. Export `useFileUpload` function returning `{ upload, progress, status, error, reset }` matching the `UseFileUploadResult` interface.
2. `upload(file: File)` does two things sequentially:
   - Step 1: `fetch('/api/upload/presign', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ filename: file.name, contentType: file.type }) })`. On non-ok response or catch, set error state and return null.
   - Step 2: Create `new XMLHttpRequest()`, add event listeners, `xhr.open('PUT', uploadUrl)`, `xhr.setRequestHeader('Content-Type', file.type)`, `xhr.send(file)`. Return a Promise that resolves to fileKey on success, null on failure.
3. `xhr.upload.addEventListener('progress', ...)` updates progress via `Math.round((e.loaded / e.total) * 100)` when `e.lengthComputable`.
4. `xhr.addEventListener('load', ...)` checks `xhr.status >= 200 && xhr.status < 300` for success. Non-2xx sets error state (Pitfall 2 — onerror does NOT fire for HTTP errors).
5. `xhr.addEventListener('error', ...)` sets network error message.
6. Error messages are exactly: presign/HTTP failure = `'Upload failed. Check your connection and try again.'`, network error = `"Connection lost. Your file wasn't uploaded — try again when you're back online."`.
