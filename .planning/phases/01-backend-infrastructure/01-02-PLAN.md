Create `cors.json` at project root with the R2 CORS policy.

**CRITICAL from research:** R2 does NOT accept `AllowedHeaders: ["*"]` — must list `"Content-Type"` explicitly. CORS must be set via Wrangler CLI, not the R2 dashboard (dashboard editor is unreliable).

For Phase 1 verification, use `AllowedOrigins: ["*"]` temporarily to unblock testing from any Vercel preview URL. This will be locked to the production domain before Phase 2 ships. (Per 01-RESEARCH.md Open Question #2 recommendation.)

```json
[
  {
    "AllowedOrigins": ["*"],
    "AllowedMethods": ["PUT"],
    "AllowedHeaders": ["Content-Type"],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3600
  }
]
```

**Note:** The exact Wrangler CORS JSON format has conflicting examples in Cloudflare docs (per 01-RESEARCH.md Open Question #1). The format above (flat array) and the wrapped `{ "rules": [...] }` format are both documented. After applying with `npx wrangler r2 bucket cors set mouse-doc-uploads --file cors.json`, immediately verify with `npx wrangler r2 bucket cors list mouse-doc-uploads`. If the flat array fails, try wrapping in `{ "rules": [...] }`.

  <action>
The user must complete these steps. Claude cannot authenticate to Cloudflare or Vercel dashboards.

**Step 1 — Authenticate Wrangler CLI:**
```bash
npx wrangler login
```
This opens a browser for Cloudflare OAuth. Complete the login flow.

**Step 2 — Create the R2 bucket:**
```bash
npx wrangler r2 bucket create mouse-doc-uploads
```
Verify:
```bash
npx wrangler r2 bucket list
```
Should show `mouse-doc-uploads` in the output.

**Step 3 — Apply CORS policy:**
```bash
npx wrangler r2 bucket cors set mouse-doc-uploads --file cors.json
```
Verify CORS was applied:
```bash
npx wrangler r2 bucket cors list mouse-doc-uploads
```
Should show the rules with `AllowedHeaders: ["Content-Type"]` and `AllowedMethods: ["PUT"]`.

If `cors list` returns empty or errors, try wrapping cors.json content in `{ "rules": [...] }` and re-run the `cors set` command.

**Step 4 — Create R2 API token:**
1. Go to Cloudflare Dashboard -> R2 Object Storage -> Manage R2 API Tokens
2. Click "Create API Token"
3. Set Permissions: "Object Read & Write"
4. Optionally scope to bucket: `mouse-doc-uploads`
5. Click Create
6. **Copy immediately** — the Secret Access Key is shown only once
7. Note all three values: Access Key ID, Secret Access Key, Account ID (visible on R2 overview page, right sidebar)

**Step 5 — Create `.env.local`:**
Copy `.env.local.example` to `.env.local` and fill in real values:
```bash
cp .env.local.example .env.local
```
Then edit `.env.local` with the real credentials from Step 4.

**Step 6 — Set Vercel environment variables:**
```bash
# If Vercel CLI is installed:
vercel env add CLOUDFLARE_ACCOUNT_ID
vercel env add R2_ACCESS_KEY_ID
vercel env add R2_SECRET_ACCESS_KEY
vercel env add R2_BUCKET_NAME
```
Or set them in the Vercel Dashboard -> Project Settings -> Environment Variables. Set for all environments (Production, Preview, Development). Mark as "Secret" type.

**Step 7 — Verify local dev server works:**
```bash
npm run dev
```
Then in a separate terminal:
```bash
curl -s -X POST http://localhost:3000/api/upload/presign \
  -H "Content-Type: application/json" \
  -d '{"filename":"test.mp4","contentType":"video/mp4"}'
```
Expected response: `{"uploadUrl":"https://....r2.cloudflarestorage.com/...","fileKey":"submissions/2026-03/..."}`.

If the response contains `uploadUrl` with a real R2 domain, credentials are working.
